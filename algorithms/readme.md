Алгоритмом называется набор инструкций для выполнений задачи. Любой фрагмент программного кода можно назвать алгоритмом.

# BIG O

> Описывает, на сколько быстро работает алгоритм

В простом поиске, чтобы найти элемент в списке размера `n` нужно совершить `n` операций - то есть **O(n)**

> О-большое не определяет время выполнения алгоритма в временном исчислении, О-большое определяет количество требуемых операций _в худшем случае_, для выполнения алгоритма

В бинарном поиске О-большое выглядит так **O(log2n)**

Типичные примеры **O-большого**: _в порядке убывания скорости выполнения_ _O(log2n) == O(log n)_

- O(log n), или логарифмическое время. Пример: бинарный поиск
- O(n), или линейное время. Пример: простой поиск
- O(n * log n). Пример: эффективные алгоритмы сортировки (быстрая сортировка)
- O(n2). Пример: медленные алгоритмы сортировки (сортировка выбором)
- O(n!). Пример: очень медленные алгоритмы
